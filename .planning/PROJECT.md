# DevOS — AI 驱动的研发操作系统

## What This Is

一个面向研发团队（产品/开发/测试/UI）的 Web 平台，通过 AI 自动将模糊需求结构化为可验证的行为模型，让团队在实现之前达成共识，从根本上减少因认知不同步导致的返工。外部部门也可提交需求并跟踪进度。

## Core Value

**在实现之前暴露误解，让团队对"要做什么"达成结构化共识。**

### Active

- [ ] 需求结构化引擎：自然语言需求 → AI 自动建模为结构化模型（目标/假设/行为/场景）
- [ ] 对话式修正：AI 生成草稿后，通过对话引导团队逐步修正和确认
- [ ] 多角色协作：产品、开发、测试、UI 各角色在同一需求模型上协作、评审、达成共识
- [ ] 外部需求入口：其他部门通过简单表单提交原始需求，可跟踪处理进度
- [ ] 知识库系统：支持上传背景文档、接入代码仓库，作为 AI 建模的上下文
- [ ] 历史需求沉淀：历史澄清过程、决策记录自动积累，系统越用越聪明
- [ ] 需求模型版本化：模型可 diff、可追溯变更历史
- [ ] 可扩展 Agent 架构：预留 AI Agent 插件接口，未来可接入实现/测试/审核 Agent

### Out of Scope

- AI 自动实现代码 — v1 聚焦需求澄清，实现能力作为后续扩展
- AI 自动测试生成 — 同上，通过 Agent 插件架构在后续版本接入
- AI 代码审核 — 同上
- 移动端 App — Web 优先
- 第三方知识库集成（飞书/Confluence）— v1 先做文档上传和代码仓库

## Context

**问题背景：**
- 团队 5-15 人，角色包括产品、开发、测试、UI
- 当前最大痛点：需求描述模糊 → 开发理解偏差 → 做完才发现不对 → 反复返工
- 四大协作问题：产品说不清楚、改了又改、测试没标准、信息断层
- AI 编码能力越来越强，但验证环节仍大量依赖人工，效率瓶颈在"方向对齐"而非"实现速度"

**设计哲学（来自原始讨论）：**
- 快速进化型，不追求一次正确，追求"模糊可控地演化"
- 像 Git 一样版本化需求模型，而非像数学证明系统
- 需求模型是中心数据结构，所有下游（实现/测试/审核）从它派生
- 系统是"刹车系统"——在高速 AI 时代防止方向失控

**需求结构化五层模型（来自原始讨论）：**
1. 目标澄清层（Goal Clarifier）— Before/After、可度量指标、冲突检测
2. 假设显性化层（Assumption Surface）— 隐性前提挖掘、置信度标注
3. 行为建模层（Behavior Modeling）— 状态图、转换规则
4. 场景枚举层（Scenario Enumeration）— 正常/异常/边界场景自动生成
5. 可验证性层（Verifiability Layer）— 哪些可自动验证、哪些需人工确认

**未来扩展方向（v2+）：**
- 验证闭环：结构化需求 → 自动生成测试 → 一键验证 → 报告
- AI 实现 Agent：从需求模型自动生成代码骨架
- 变更影响预测：模型 diff → 自动预测受影响模块和测试

## Constraints

- **产品形态**: Web 应用
- **技术栈**: 待研究阶段确定
- **架构**: 事件驱动 + 插件化 Agent 接口，确保可扩展性
- **用户规模**: 5-15 人研发团队 + 外部需求提交者
- **定位**: 内部生产力工具（自用），非商业化产品
