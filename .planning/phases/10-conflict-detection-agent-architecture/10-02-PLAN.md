# Plan 10-02: Agent Plugin Interface Extraction

## Goal

Extract the already-working clarifier flow into a reusable agent plugin contract, then prove the abstraction by adding a second concrete agent for conflict detection.

## Scope

1. Define shared agent types and execution context.
2. Implement a central registry with register/get/list/run semantics.
3. Register the existing clarifier as a first-class agent.
4. Register the conflict detector as a second agent using the same interface.
5. Route `structure` and `conflict` execution through the registry.
6. Add basic registry tests.

## Done When

- Agent behavior is discoverable from one registry, not hardcoded across routes.
- Two concrete agents (`clarifier`, `conflict-detector`) prove the interface is not single-use.
- New agents can be added by implementing one typed contract and registering once.
