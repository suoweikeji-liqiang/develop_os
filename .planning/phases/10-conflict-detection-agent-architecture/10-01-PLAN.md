# Plan 10-01: Cross-Requirement Conflict Detection Engine

## Goal

Add persisted conflict detection that can flag contradictions within a requirement model and between neighboring requirements, then expose those results through a reviewable UI.

## Scope

1. Add `RequirementConflict` storage and supporting enums to Prisma.
2. Define a typed conflict schema for AI output.
3. Build the conflict scan service that loads the current requirement plus nearby candidates, runs AI detection, fingerprints results, and upserts rows.
4. Wire conflict APIs into tRPC and auto-trigger scans after model updates.
5. Add a requirement detail panel for list / rescan / dismiss / resolve workflows.

## Done When

- Conflict rows are stored in PostgreSQL and deduplicated by fingerprint.
- The requirement detail page shows open, dismissed, and resolved conflicts with evidence and suggested action.
- Manual rescan and status updates are available to authenticated users.
- Cross-requirement contradictions survive page refresh because they are persisted, not transient.
