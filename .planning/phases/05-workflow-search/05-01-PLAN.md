   - Add indexes: `@@index([status])`, `@@index([createdBy])`, `@@index([createdAt])`

   - IMPORTANT: The migration will likely need manual SQL editing because existing rows have lowercase "draft". If Prisma generates a migration that fails, edit the SQL to: (1) UPDATE all existing rows SET status = 'DRAFT' WHERE status = 'draft', (2) then CREATE TYPE and ALTER COLUMN. Alternatively, if no data exists yet in dev, a clean migration is fine.
   - Run `npx prisma generate` to regenerate client

2. **Status machine (src/lib/workflow/status-machine.ts):**
   - Create Zod enum `RequirementStatusEnum` with z.enum(['DRAFT', 'IN_REVIEW', 'CONSENSUS', 'IMPLEMENTING', 'DONE'])
   - Export type `RequirementStatus = z.infer<typeof RequirementStatusEnum>`
   - Define `TRANSITIONS: Record<RequirementStatus, readonly RequirementStatus[]>` map:
     - DRAFT: ['IN_REVIEW']
     - IN_REVIEW: ['CONSENSUS', 'DRAFT']
     - CONSENSUS: ['IMPLEMENTING', 'IN_REVIEW']
     - IMPLEMENTING: ['DONE', 'CONSENSUS']
     - DONE: [] (terminal)
   - Export `canTransition(from, to): boolean`
   - Export `getValidTransitions(current): readonly RequirementStatus[]`
   - Export `assertTransition(from, to): void` — throws Error with message `Invalid transition: ${from} -> ${to}`

3. **Status labels (src/lib/workflow/status-labels.ts):**
   - Import RequirementStatus type from status-machine
   - Export `STATUS_LABELS: Record<RequirementStatus, string>` — DRAFT:'草稿', IN_REVIEW:'评审中', CONSENSUS:'共识达成', IMPLEMENTING:'实现中', DONE:'完成'
   - Export `STATUS_COLORS: Record<RequirementStatus, string>` — Tailwind classes: DRAFT:'bg-gray-100 text-gray-700', IN_REVIEW:'bg-yellow-100 text-yellow-800', CONSENSUS:'bg-blue-100 text-blue-800', IMPLEMENTING:'bg-purple-100 text-purple-800', DONE:'bg-green-100 text-green-800'

4. **Event type (src/server/events/types.ts):**
   - Add `'requirement.status.changed': { requirementId: string; from: string; to: string; changedBy: string }` to EventMap

5. **tRPC mutation (src/server/trpc/routers/requirement.ts):**
   - Import `assertTransition` and `RequirementStatusEnum` from '@/lib/workflow/status-machine'
   - Add `transitionStatus` mutation to requirementRouter:
     - Input: `z.object({ id: z.string(), to: RequirementStatusEnum })`
     - Fetch current requirement status with `findUniqueOrThrow`
     - Call `assertTransition(current.status as RequirementStatus, input.to)` — wrap in try/catch, throw TRPCError BAD_REQUEST on invalid transition
     - Update with optimistic lock: `prisma.requirement.update({ where: { id: input.id, status: current.status }, data: { status: input.to } })`
     - Emit `requirement.status.changed` event with from/to/changedBy
     - Return updated requirement
   - Also update the existing `list` procedure's input to accept `RequirementStatusEnum.optional()` instead of `z.string().optional()` for type safety
