  <files>

  <action>
1. **Add `search` procedure to requirementRouter:**
   - Import `RequirementStatusEnum` from '@/lib/workflow/status-machine'
   - Input schema: `z.object({ query: z.string().optional(), status: RequirementStatusEnum.optional(), tags: z.array(z.string()).optional(), dateFrom: z.coerce.date().optional(), dateTo: z.coerce.date().optional() }).optional()`
   - Build Prisma `where` clause using AND array:
     - Text search: if `query`, use `OR: [{ title: { contains: query, mode: 'insensitive' } }, { rawInput: { contains: query, mode: 'insensitive' } }]`
     - Status filter: if `status`, add `{ status }`
     - Tag filter: if `tags?.length`, add `{ tags: { hasSome: tags } }`
     - Date from: if `dateFrom`, add `{ createdAt: { gte: dateFrom } }`
     - Date to: if `dateTo`, add `{ createdAt: { lte: dateTo } }`
   - Return `prisma.requirement.findMany({ where, orderBy: { updatedAt: 'desc' }, select: { id, title, rawInput, status, tags, createdBy, createdAt, updatedAt, version } })`

2. **Add `addTag` mutation:**
   - Input: `z.object({ id: z.string(), tag: z.string().min(1).max(50) })`
   - Use `prisma.requirement.update({ where: { id }, data: { tags: { push: tag } } })`
   - Return updated requirement

3. **Add `removeTag` mutation:**
   - Input: `z.object({ id: z.string(), tag: z.string() })`
   - Fetch current tags, filter out the target tag, update with new array
   - Return updated requirement

Note: The existing `list` procedure can remain for backward compatibility, but the new `search` procedure is the primary query for the requirements list page.
