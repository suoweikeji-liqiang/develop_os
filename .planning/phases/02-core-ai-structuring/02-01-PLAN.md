  <files>src/lib/schemas/requirement.ts, prisma/schema.prisma</files>

1. Create `src/lib/schemas/requirement.ts` with the FiveLayerModelSchema:
   - Import from `zod` (project uses `import { z } from 'zod'` pattern, NOT `zod/v4`)
   - Define schema with 5 top-level keys: goal, assumption, behavior, scenario, verifiability
   - goal: `{ summary: string, before: string, after: string, metrics: string[] }`
   - assumption: `{ items: Array<{ content: string, confidence: enum('high','medium','low'), rationale: string }> }`
   - behavior: `{ actors: string[], actions: Array<{ actor: string, action: string, precondition?: string, postcondition?: string }> }`
   - scenario: `{ normal: Array<{ name: string, steps: string[] }>, edge: Array<{ name: string, steps: string[], trigger: string }>, error: Array<{ name: string, steps: string[], recovery: string }> }`
   - verifiability: `{ automated: Array<{ criterion: string, method: string }>, manual: Array<{ criterion: string, reason: string }> }`
   - Use `.describe()` on every field (English descriptions — these guide the LLM)
   - Export the schema AND the inferred TypeScript type: `export type FiveLayerModel = z.infer<typeof FiveLayerModelSchema>`
   - Also export a `CreateRequirementSchema` for the tRPC create mutation: `{ title: string, rawInput: string }`

2. Evolve `prisma/schema.prisma` Requirement model — add fields:
   - `rawInput  String  @db.Text` (original user input, required)
   - `confidence Json?` (per-layer confidence scores)
   - `attempts  Int     @default(1)` (AI generation attempt count)
   - `version   Int     @default(1)` (version tracking prep for Phase 4)
   - Keep existing fields unchanged

3. Run `npx prisma migrate dev --name add-requirement-ai-fields` to apply migration.
4. Run `npx prisma generate` to regenerate client.
  </action>
