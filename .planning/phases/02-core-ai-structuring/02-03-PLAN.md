<task type="auto">
  <name>Task 2: Build tab-based model display with streaming and inline editing</name>
  <files>src/app/(dashboard)/requirements/[id]/page.tsx, src/app/(dashboard)/requirements/[id]/model-tabs.tsx, src/app/(dashboard)/requirements/[id]/layer-editor.tsx</files>
  <action>
1. Create `src/app/(dashboard)/requirements/[id]/page.tsx`:
   - Server component with `export const dynamic = 'force-dynamic'`
   - Fetch requirement by ID using Prisma directly (import prisma from @/server/db/client)
   - If not found, call notFound()
   - If requirement has no model yet (model is null), render the ModelTabs client component in "generate" mode passing rawInput and requirementId
   - If requirement has model, render ModelTabs in "view" mode passing the existing model data
   - Show requirement title and status at the top

2. Create `src/app/(dashboard)/requirements/[id]/model-tabs.tsx`:
   - "use client" component
   - Props: `{ requirementId: string, rawInput: string, initialModel?: FiveLayerModel, mode: 'generate' | 'view' }`
   - Use shadcn/ui Tabs component (install if needed: `npx shadcn@latest add tabs`)
   - 5 tabs: 目标 (Goal), 假设 (Assumption), 行为 (Behavior), 场景 (Scenario), 可验证性 (Verifiability)
   - Generate mode: Import `useObject` from `ai/react`, call with `{ api: '/api/ai/structure', schema: FiveLayerModelSchema }`. On mount call `submit({ rawInput })`. Display partial data with optional chaining. Show Skeleton loaders for incomplete layers. On completion persist via tRPC `requirement.updateModel`. On error show message with attempt count.
   - View mode: Display initialModel in tabs, each tab renders LayerEditor for inline editing.
   - Confidence badges: Badge (shadcn/ui) per layer — high=green, medium=yellow, low=red
   - Install shadcn components: `npx shadcn@latest add tabs badge skeleton`

3. Create `src/app/(dashboard)/requirements/[id]/layer-editor.tsx`:
   - "use client" component
   - Props: `{ layerName: string, data: object, requirementId: string, onUpdate: (data) => void }`
   - Render layer fields as editable inputs/textareas based on layer structure
   - Goal: summary textarea, before/after textareas, metrics as editable list
   - Assumption: items with content textarea, confidence dropdown, rationale textarea
   - Behavior: actors list, actions table with actor/action/pre/postcondition
   - Scenario: grouped by normal/edge/error, each with name + steps list + trigger/recovery
   - Verifiability: automated and manual lists with criterion + method/reason
   - On blur or explicit save, call `onUpdate` with modified layer data
   - Use shadcn/ui: Input, Textarea, Select, Button. Chinese labels for all field names.
  </action>
