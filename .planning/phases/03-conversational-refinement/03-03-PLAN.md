# Plan 03-03: Chat Panel UI and Assumption Surfacing

## Goal

Build the chat panel component using `useChat`, the assumption card component with Accept/Reject/Edit actions, and wire both into the requirement detail page. The chat panel opens automatically after initial model generation and is collapsible.

## Requirements

- AI-02: 用户可通过对话方式修正和精炼 AI 生成的结构化模型
- AI-03: AI 自动识别需求中的隐含假设并标注置信度

## must_haves

- `ChatPanel` component renders chat bubbles (user right, AI left), a textarea input, and a send button
- `useChat` is initialized with `initialMessages` from DB and `transport: new DefaultChatTransport({ api: '/api/ai/converse', body: { requirementId, currentModel } })`
- `onFinish` callback saves the completed assistant message to DB via tRPC `conversation.saveMessage` mutation
- Chat panel auto-opens when `initialMessages.length > 0` or when `autoOpen` prop is true; has a collapse toggle button
- Loading skeleton shown while messages are loading (tRPC `getMessages` query)
- "Load more" button appears when `hasMore` is true from paginated query
- `AssumptionCard` component renders with amber glow (`ring-2 ring-amber-400 bg-amber-50`), shows `content`, `confidence` badge, and `rationale`
- `AssumptionCard` has three action states: Accept (shows inline edit textarea pre-filled with content, confirm button), Reject (shows small textarea for reason, confirm button), Edit (same as Accept)
- After action is confirmed, card transitions to normal appearance (ring/bg removed) and calls `onAction` callback with `{ type: 'accept' | 'reject', finalContent?: string, rejectReason?: string }`
- Requirement detail page (`page.tsx`) loads last 20 conversation messages server-side and passes as `initialMessages` to `ChatPanel`
- Page layout changes to side-by-side: model tabs on left, chat panel on right (`grid grid-cols-[1fr_400px]` when chat is open, single column when collapsed)
- Chat input is disabled while `status === 'streaming'` from `useChat`

## Context

### Locked decisions (NON-NEGOTIABLE)

- Side-by-side layout: chat right, model tabs left
- Chat bubble style: user on right, AI on left
- Chat panel auto-opens after initial model generation AND when returning to a requirement with existing messages
- Assumption cards appear on the Assumption tab with amber glow
- Each assumption card: Accept / Reject / Edit buttons
- Accept allows inline editing before confirming
- Reject opens small text field for reason
- After action, card blends into normal assumption list

      `useChat` setup:
      ```typescript
      const { messages, sendMessage, status, error } = useChat({
        messages: initialMessages,
        transport: new DefaultChatTransport({
          api: '/api/ai/converse',
          body: { requirementId, currentModel },
        }),
        onFinish: async ({ message }) => {
          // 1. Save assistant message to DB
          await fetch('/api/trpc/conversation.saveMessage', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
              json: { requirementId, role: 'assistant', content: message.parts ?? [] }
            }),
          })
          // 2. Extract structured response from message parts
          // Parse ConversationResponse from the text content part — wrap in try/catch
          try {
            const textPart = message.parts?.find((p: { type: string }) => p.type === 'text')
            if (textPart && 'text' in textPart) {
              const parsed = JSON.parse((textPart as { text: string }).text) as ConversationResponse
              if (parsed.patches || parsed.newAssumptions?.length) {
                onPatchProposed(parsed)
              }
              if (parsed.newAssumptions?.length) {
                setPendingAssumptions(prev => [
                  ...prev,
                  ...parsed.newAssumptions!.map(a => ({
                    id: crypto.randomUUID(),
                    ...a,
                  }))
                ])
              }
            }
          } catch {
            // parse failure — skip silently
          }
        },
      })
      ```

      Auto-scroll: use a `messagesEndRef` div at the bottom of the message list, call `scrollIntoView({ behavior: 'smooth' })` in a `useEffect` on `messages` change.

      Send handler:
      ```typescript
      async function handleSend() {
        if (!inputValue.trim() || status === 'streaming') return
        const text = inputValue.trim()
        setInputValue('')
        // Save user message to DB
        await fetch('/api/trpc/conversation.saveMessage', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({ json: { requirementId, role: 'user', content: [{ type: 'text', text }] } }),
        })
        sendMessage({ role: 'user', content: text })
      }
      ```

      Keyboard handler: `onKeyDown` — send on Enter (without Shift), allow Shift+Enter for newline.
