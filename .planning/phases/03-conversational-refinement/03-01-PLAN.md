# Plan 03-01: Conversation Schema and Prompt Builder

## Goal

Lay the data and AI foundation for multi-turn conversation: add the `ConversationMessage` Prisma model and run migration, define the `ConversationResponseSchema` Zod schema, and build the system prompt builder function.

## Requirements

- AI-02: 用户可通过对话方式修正和精炼 AI 生成的结构化模型
- AI-03: AI 自动识别需求中的隐含假设并标注置信度

## must_haves

- `ConversationMessage` Prisma model exists with `requirementId`, `role`, `content` (Json), `createdAt`, and a composite index on `[requirementId, createdAt]`
- `Requirement` model in schema has `conversations ConversationMessage[]` relation
- `ConversationResponseSchema` Zod schema exports `reply`, `patches` (optional, partial FiveLayerModel per layer), `newAssumptions` (optional array with `content`, `confidence`, `rationale`), and `affectedLayers` (optional array)
- `buildConversationPrompt(currentModel)` returns an English system prompt that instructs the AI to: populate `patches` only for changed layers, omit `newAssumptions` unless a genuinely new implicit constraint is revealed, match output language to user input language
- Prisma migration runs without errors (`prisma migrate dev`)
- A developer can open the requirement detail page and see the chat panel render (end-to-end smoke test passes once 03-02 and 03-03 are complete — these artifacts are the prerequisite)

## Context

### Existing patterns to follow

- Prisma 7 requires explicit relation on both sides — add `conversations ConversationMessage[]` to `Requirement` model
- `Output.object` with native Zod 4 schema — no `jsonSchema()` bridge needed (verified in Phase 2)
- Import `FiveLayerModelSchema` from `@/lib/schemas/requirement` and use `.shape` to access individual layer schemas

### FiveLayerModelSchema shape (for patches)

```typescript
// src/lib/schemas/requirement.ts — already exists
// patches uses optional versions of each layer shape:
patches: z.object({
  goal: GoalSchema.optional(),
  assumption: AssumptionSchema.optional(),
  behavior: BehaviorSchema.optional(),
  scenario: ScenarioSchema.optional(),
  verifiability: VerifiabilitySchema.optional(),
}).optional()
```

      Create new file. Import `z` from `zod` and `FiveLayerModelSchema` from `@/lib/schemas/requirement`.

      Define `ModelPatchSchema` using optional versions of each layer from `FiveLayerModelSchema.shape`:
      ```typescript
      const ModelPatchSchema = z.object({
        goal: FiveLayerModelSchema.shape.goal.optional(),
        assumption: FiveLayerModelSchema.shape.assumption.optional(),
        behavior: FiveLayerModelSchema.shape.behavior.optional(),
        scenario: FiveLayerModelSchema.shape.scenario.optional(),
        verifiability: FiveLayerModelSchema.shape.verifiability.optional(),
      })
      ```

      Define and export `ConversationResponseSchema`:
      ```typescript
      export const ConversationResponseSchema = z.object({
        reply: z.string().describe('Conversational response shown in chat bubble'),
        patches: ModelPatchSchema.optional().describe('Model layer updates proposed by AI — only include layers that changed'),
        newAssumptions: z.array(z.object({
          content: z.string(),
          confidence: z.enum(['high', 'medium', 'low']),
          rationale: z.string(),
        })).optional().describe('New implicit assumptions surfaced — omit if no genuinely new constraints revealed'),
        affectedLayers: z.array(
          z.enum(['goal', 'assumption', 'behavior', 'scenario', 'verifiability'])
        ).optional().describe('Which layers are affected by this response'),
      })

      export type ConversationResponse = z.infer<typeof ConversationResponseSchema>
      ```
    </action>
