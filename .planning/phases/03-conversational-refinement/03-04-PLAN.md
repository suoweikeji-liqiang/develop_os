# Plan 03-04: Model Update Pipeline (Diff, Confirm, Undo, Assumption Integration)

## Goal

Wire the full model update pipeline: AI-proposed patches render as inline color diffs on ModelTabs/LayerEditor, user confirms or rejects per layer, one-step undo is available after applying, and surfaced assumptions appear as AssumptionCards on the Assumption tab.

## Requirements

- AI-02: 用户可通过对话方式修正和精炼 AI 生成的结构化模型
- AI-03: AI 自动识别需求中的隐含假设并标注置信度

## must_haves

- When AI returns `patches`, `ModelTabs` enters diff mode: affected tabs show a visual indicator (amber dot badge)
- `LayerEditor` accepts a `pendingData` prop; when present, renders changed fields with green (added/changed) and red (removed) backgrounds side-by-side
- Per-layer confirm/reject buttons appear in diff mode — user can accept or reject each layer's changes independently
- After all pending diffs are resolved (accepted or rejected), diff mode exits
- Accepted changes call `requirement.updateModel` tRPC mutation (via the existing `persistModel` fetch pattern)
- One-step undo button appears after applying any change; clicking it reverts `model` state to the pre-apply snapshot (component state only, no DB write for undo)
- `newAssumptions` from AI response are injected into the Assumption tab as `AssumptionCard` components above the existing assumption list
- Chat input is disabled while any pending diff exists (show "请先确认或拒绝建议的变更" message)
- `ChatPanel` `onPatchProposed` callback is fully wired in `page.tsx` via `RequirementDetailClient` — it updates shared state that flows into `ModelTabs` as `pendingPatches` and into `ChatPanel` as a `hasPendingDiff` flag
- A user can send a chat message, see the AI diff appear on the model tabs, click "接受变更", and see the model update — the full round-trip works end-to-end

## Context

### State management approach

The requirement detail page (`page.tsx`) is a server component — it cannot hold client state. The solution is to extract a new client wrapper component that holds the shared state between `ChatPanel` and `ModelTabs`.

Create `src/app/(dashboard)/requirements/[id]/requirement-detail-client.tsx` as a `'use client'` component that:
- Receives `requirement` data and `initialMessages` as props
- Holds `pendingPatches` state (from AI response)
- Holds `pendingAssumptions` state (from AI response)
- Holds `previousModel` ref (for undo)
- Renders the side-by-side layout with `ModelTabs` and `ChatPanel`

`page.tsx` becomes a thin server component that fetches data and renders `RequirementDetailClient`.

### Diff rendering approach

Field-level comparison — no external library. For each field in a layer:
- If `current === proposed`: render normally
- If `current !== proposed`: render old value with `bg-red-100 line-through text-red-700` and new value with `bg-green-100 text-green-700`
- For arrays: compare item by item (by index); added items get green bg, removed items get red bg with strikethrough

`LayerEditor` gets two new optional props:
```typescript
pendingData?: Record<string, unknown>  // proposed by AI
onConfirmDiff?: () => void
onRejectDiff?: () => void
```

When `pendingData` is present, the editor renders in read-only diff mode (no editing allowed while diff is pending).

### Undo state

```typescript
const previousModelRef = useRef<FiveLayerModel | null>(null)
const [showUndo, setShowUndo] = useState(false)

function applyPatch(layer: LayerKey, proposedData: unknown) {
  previousModelRef.current = currentModel  // snapshot before apply
  setShowUndo(true)
  // apply the change and persist
}

function handleUndo() {
  if (previousModelRef.current) {
    setModel(previousModelRef.current)
    setShowUndo(false)
    previousModelRef.current = null
    // No DB write — undo is ephemeral
  }
}
```

### Assumption injection

`ModelTabs` gets a new prop `pendingAssumptions: PendingAssumption[]`. On the Assumption tab, render `AssumptionCard` components above the existing `LayerEditor`. When a card action is confirmed (accept), add the assumption to the model's assumption layer and persist. When rejected, remove from `pendingAssumptions` state.

### Chat input blocking

`ChatPanel` gets a `hasPendingDiff: boolean` prop. When true:
- Disable the textarea and send button
- Show a small message below the input: "请先确认或拒绝建议的变更"

### persistModel fetch pattern (from model-tabs.tsx)

```typescript
// Follow the existing persistModel pattern in model-tabs.tsx for all DB writes

  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ json: { id: requirementId, model: updatedModel } }),
})
```

      ```typescript
      const handleApplyPatch = useCallback(async (layer: string, proposedData: unknown) => {

        const updatedModel = model ? { ...model, [layer]: proposedData } as FiveLayerModel : undefined

        if (updatedModel) {

        setPendingPatches(prev => {
          if (!prev) return null

          return Object.keys(next).length === 0 ? null : next
      ```

      ```typescript
      function DiffSummary({ current, proposed }: { current: Record<string, unknown>; proposed: Record<string, unknown> }) {
        const allKeys = new Set([...Object.keys(current), ...Object.keys(proposed)])
        const changedKeys = [...allKeys].filter(k => JSON.stringify(current[k]) !== JSON.stringify(proposed[k]))

        if (changedKeys.length === 0) return <p className="text-sm text-muted-foreground">无变更</p>

        return (
          <div className="space-y-1">
            {changedKeys.map(key => (
              <DiffField key={key} fieldName={key} current={current[key]} proposed={proposed[key]} />
            ))}
          </div>
        )
      }

      function DiffField({ fieldName, current, proposed }: { fieldName: string; current: unknown; proposed: unknown }) {
        const currentStr = typeof current === 'string' ? current : JSON.stringify(current)
        const proposedStr = typeof proposed === 'string' ? proposed : JSON.stringify(proposed)

        return (
          <div className="text-xs space-y-0.5">
            <span className="font-medium text-muted-foreground">{fieldName}: </span>
            {current !== undefined && (
              <span className="bg-red-100 line-through text-red-700 px-1 rounded">{currentStr}</span>
            )}
            {' → '}
            {proposed !== undefined && (
              <span className="bg-green-100 text-green-700 px-1 rounded">{proposedStr}</span>
            )}
          </div>
        )
      }
      ```
