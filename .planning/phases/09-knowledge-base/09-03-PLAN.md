- CODE_EXTENSIONS: ['.ts', '.tsx', '.js', '.jsx', '.py', '.go', '.java', '.md']
- MAX_FILE_SIZE: 50_000 bytes
- File content is base64-encoded in Octokit response

- GitHub tokens MUST be stored encrypted
- Use Node.js built-in `crypto` module — `createCipheriv` / `createDecipheriv`
- Require `ENCRYPTION_KEY` env var (32-byte hex string)
- Never expose `githubTokenEncrypted` to client via tRPC

Rate limiting constraint (RESEARCH.md Pitfall 5):
- Process files in batches of 20-30 with delay between batches
- Handle HTTP 403/429 gracefully

  <name>Task 1: AES-256 token encryption utility + GitHub repository sync</name>
  <files>

  <action>
    Create `src/server/ai/rag/crypto.ts`:
    - Import `{ createCipheriv, createDecipheriv, randomBytes } from 'crypto'`
    - `ENCRYPTION_KEY` from `process.env.ENCRYPTION_KEY` — validate it exists and is 64 hex chars (32 bytes) at module load time; throw clear error if missing
    - `encryptToken(plaintext: string): string` — use AES-256-CBC with a random 16-byte IV; return `${iv.toString('hex')}:${encrypted.toString('hex')}`
    - `decryptToken(encrypted: string): string` — split on ':', reconstruct IV and cipher text, return decrypted UTF-8 string
    - Both functions are pure and synchronous

    Create `src/server/ai/rag/github.ts`:
    - Import `{ Octokit } from '@octokit/rest'`
    - Import `{ chunkText, embedAndStore } from './embed'`
    - Import `{ prisma } from '@/server/db/client'`
    - Export `CODE_EXTENSIONS = ['.ts', '.tsx', '.js', '.jsx', '.py', '.go', '.java', '.md', '.prisma', '.sql', '.yaml', '.json']`
    - Export `MAX_FILE_SIZE = 50_000`
    - Export `BATCH_SIZE = 20`

    Export `fetchAndEmbedRepository(repositoryId: string, owner: string, repo: string, githubToken: string): Promise<void>`:
    1. Create Octokit with `auth: githubToken`
    2. Get repo default branch via `octokit.repos.get({ owner, repo })`
    3. Get full recursive tree via `octokit.git.getTree({ owner, repo, tree_sha: branch, recursive: 'true' })`
    4. Filter files: `type === 'blob'` AND extension in CODE_EXTENSIONS AND `(size ?? 0) < MAX_FILE_SIZE`
    5. Before embedding, delete existing chunks for this repo: `prisma.$executeRaw\`DELETE FROM "KnowledgeChunk" WHERE "sourceType" = 'code' AND "sourceId" = ${repositoryId}\``
    6. Process files in batches of BATCH_SIZE:
       - For each file: `octokit.repos.getContent({ owner, repo, path: file.path! })`
       - Decode: `Buffer.from((data as any).content, 'base64').toString('utf-8')`
       - `const chunks = chunkText(content)`
       - `await embedAndStore(chunks, 'code', repositoryId, { sourceName: file.path! })`
       - After each batch: `await new Promise(r => setTimeout(r, 1000))` (1s delay for rate limiting)
    7. On completion: `prisma.codeRepository.update({ where: { id: repositoryId }, data: { lastSyncedAt: new Date() } })`
    8. Wrap all Octokit calls in try/catch — on HTTP 403/429, log error and throw; caller handles gracefully
  </action>

  </verify>

    `add(input: AddCodeRepositoryInputSchema)`:
    - Encrypt the GitHub token: `const encrypted = encryptToken(input.githubToken)`
    - `prisma.codeRepository.create({ data: { owner, repo, githubTokenEncrypted: encrypted, addedBy: ctx.session.userId } })`
    - Fire-and-forget sync after creation (matches fire-and-forget pattern):
      `void fetchAndEmbedRepository(created.id, owner, repo, input.githubToken).catch(console.error)`
    - Return `{ id: created.id, owner, repo, lastSyncedAt: null }` (never expose encrypted token)

    `list`:
    - `prisma.codeRepository.findMany({ where: { addedBy: ctx.session.userId }, orderBy: { createdAt: 'desc' }, select: { id: true, owner: true, repo: true, lastSyncedAt: true, createdAt: true } })`
    - select explicitly excludes `githubTokenEncrypted`

    `sync(id: z.string())`:
    - Fetch repo with ownership check
    - Decrypt token: `const token = decryptToken(repo.githubTokenEncrypted)`
    - Fire-and-forget: `void fetchAndEmbedRepository(id, repo.owner, repo.repo, token).catch(console.error)`
    - Return `{ syncing: true }`

    `delete(id: z.string())`:
    - Verify ownership
    - `prisma.$executeRaw\`DELETE FROM "KnowledgeChunk" WHERE "sourceType" = 'code' AND "sourceId" = ${id}\``
