Create `prisma/schema.prisma` with the following models per research and user decisions:

**datasource:** postgresql, env("DATABASE_URL")
**generator:** prisma-client-js

**Models (use exact schema from research):**

1. `User` — id (cuid), email (unique), password, name, isAdmin (default false), relations to UserRole[] and Session[], createdAt, updatedAt
2. `UserRole` — id (cuid), userId, role (Role enum), relation to User with onDelete Cascade. Unique constraint on [userId, role]. Index on userId.
3. `Role` enum — PRODUCT, DEV, TEST, UI, EXTERNAL
4. `Session` — id (cuid), userId, expiresAt, relation to User with onDelete Cascade, createdAt. Index on userId.
5. `Invite` — id (cuid), email, roles (Role[]), token (unique), invitedBy, expiresAt, usedAt (optional), createdAt. Index on token.
6. `Requirement` — id (cuid), title, model (Json? for five-layer JSONB), status (default "draft"), createdBy, createdAt, updatedAt.

Run migrations:
```bash
npx prisma migrate dev --name init
```

Create `src/server/db/client.ts` — Prisma client singleton using the globalThis pattern to prevent hot-reload connection exhaustion:
```typescript
import { PrismaClient } from '@prisma/client'

const globalForPrisma = globalThis as unknown as { prisma: PrismaClient }
export const prisma = globalForPrisma.prisma || new PrismaClient()
if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
```

Create `src/lib/definitions.ts` with shared Zod schemas:
- `LoginSchema` — email (z.string().email()), password (z.string().min(1))
- `RegisterSchema` — email, password (min 8), name (min 1)
- `InviteSchema` — email, roles (z.array of Role enum values)
- Export the Role enum values as a const array for reuse
