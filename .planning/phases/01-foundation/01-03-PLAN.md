```typescript
export interface EventMap {
  'user.registered': { userId: string; email: string }
  'user.invited': { inviteId: string; email: string; roles: string[] }
  'user.role.assigned': { userId: string; role: string; assignedBy: string }
  'user.role.removed': { userId: string; role: string; removedBy: string }
  'session.created': { sessionId: string; userId: string }
  'session.deleted': { sessionId: string; userId: string }
  'requirement.created': { requirementId: string; createdBy: string }
}
```

**`src/server/events/bus.ts`:**
Create EventEmitter3 singleton with the typed EventMap. Follow research Pattern 3 exactly:

```typescript
import EventEmitter from 'eventemitter3'
import type { EventMap } from './types'

export const eventBus = new EventEmitter<EventMap>()
```

Use the globalThis singleton pattern (same as Prisma client) to prevent duplicate instances during hot reload:

```typescript
const globalForEvents = globalThis as unknown as { eventBus: EventEmitter<EventMap> }
export const eventBus = globalForEvents.eventBus || new EventEmitter<EventMap>()
if (process.env.NODE_ENV !== 'production') globalForEvents.eventBus = eventBus
```

This is the complete event bus. Per user decision, it is synchronous in-process EventEmitter3 — no async message queue, no external broker. Listeners will be registered at module initialization in future plans, not per-request (per research anti-pattern guidance).
