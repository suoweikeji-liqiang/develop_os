  <action>
Create the server-side auth modules. All files must import `server-only` at the top.

**`src/server/auth/password.ts`:**
- `hashPassword(password: string)` — bcrypt hash with 10 salt rounds
- `verifyPassword(password: string, hash: string)` — bcrypt compare

**`src/server/auth/session.ts`:**
Follow the exact pattern from research (Pattern 1). Use jose for session encryption.
- `createSession(userId: string)` — Create DB session (7-day expiry), encrypt session ID with jose HS256 JWT, set HttpOnly cookie named "session" with sameSite: "lax", secure in production, path "/"
- `deleteSession()` — Read cookie, decrypt, delete session from DB, delete cookie
- `getSession()` — Read cookie, decrypt, look up session in DB (include user with roles), return null if expired or not found. Do NOT redirect — let callers decide.

Secret key from `process.env.SESSION_SECRET` encoded with TextEncoder.

**`src/server/auth/invite.ts`:**
- `createInvite(email: string, roles: Role[], invitedBy: string)` — Generate 32-byte hex token via `crypto.randomBytes`, create Invite record with 48-hour expiry, return the token. Console.log the invite URL for development (no real email sending in Phase 1).
- `validateInvite(token: string)` — Find invite by token, check not expired, check not used. Return invite or null.
- `markInviteUsed(inviteId: string)` — Set usedAt to now.

**`src/lib/dal.ts`:**
Follow research Pattern 2. Use React `cache()` for request deduplication.
- `verifySession()` — Call getSession(), if null redirect to /login. Return `{ userId, roles, isAdmin }`.

**Server Actions (`src/app/actions/auth.ts`):**
Use `'use server'` directive. Follow research code examples closely.

- `login(prevState, formData)` — Validate with LoginSchema, find user by email, verify password with bcrypt, create session, redirect to /dashboard. Return `{ error: string }` on failure.
- `registerFirstUser(prevState, formData)` — Use Prisma transaction to check user count is 0 (race condition protection per research Pitfall 1). Hash password, create user with isAdmin: true, create session, redirect to /dashboard.
- `registerWithInvite(prevState, formData)` — Validate invite token from hidden field, hash password, create user, create UserRole records for each role in invite, mark invite used, create session, redirect to /dashboard.
- `logout()` — Delete session, redirect to /login.

**tRPC setup (`src/server/trpc/init.ts`):**
- Create context with session from getSession()
- `baseProcedure` — no auth required
- `protectedProcedure` — requires valid session (throw UNAUTHORIZED if no session)
- `adminProcedure` — extends protectedProcedure, requires isAdmin (throw FORBIDDEN if not admin)

**tRPC root router (`src/server/trpc/router.ts`):**
- Merge auth and user routers

**Auth router (`src/server/trpc/routers/auth.ts`):**
- `me` query — return current user info (id, email, name, roles, isAdmin) via protectedProcedure

**User router (`src/server/trpc/routers/user.ts`):**
Follow research code example for role management.
- `list` query (adminProcedure) — list all users with their roles
- `assignRole` mutation (adminProcedure) — add role to user via UserRole create, input: { userId, role }
- `removeRole` mutation (adminProcedure) — remove role from user via UserRole delete, input: { userId, role }
- `sendInvite` mutation (adminProcedure) — call createInvite, input: { email, roles }

**tRPC API route (`src/app/api/trpc/[trpc]/route.ts`):**
- Standard Next.js App Router handler for tRPC

**Auth pages (Chinese UI labels, English code):**

`src/app/(auth)/login/page.tsx` — Form with email/password fields, submit calls login Server Action. Use `useActionState`. Show error message. Link to /register. Labels in Chinese: "邮箱", "密码", "登录".

`src/app/(auth)/register/page.tsx` — First-user bootstrap form. Check if any users exist (server-side). If users exist, show "仅限邀请注册" message. If no users, show registration form (name, email, password). Labels: "姓名", "邮箱", "密码", "注册".

`src/app/(auth)/invite/[token]/page.tsx` — Validate token server-side. If invalid/expired, show error. If valid, show registration form with hidden token field. Call registerWithInvite.

**Dashboard pages:**

`src/app/(dashboard)/layout.tsx` — Call verifySession(). Show user info and logout button in header. Pass session to children via context or props.

`src/app/(dashboard)/page.tsx` — Simple dashboard placeholder showing "欢迎使用 DevOS" and user's roles.

`src/app/(dashboard)/admin/users/page.tsx` — Admin-only page (check isAdmin, redirect if not). List users with their roles. For each user: dropdown to add role, button to remove role. Invite form at top: email + role checkboxes + send button. Use tRPC mutations for all actions.
